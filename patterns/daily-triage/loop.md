# Daily Triage Loop

## Purpose
Summarize what changed, identify what needs attention, and recommend the next action.

## Cadence
Daily or manual.

## Inputs
- STATE.md
- GitHub issues / PRs
- Recent commits
- Notes from user

## Steps
1. Read project state.
2. Find recent changes.
3. Group items into urgent, waiting, blocked, and FYI.
4. Recommend the top 3 actions.
5. Update run log.

## Output
A short daily report.

## Safety
Report-only by default.
